# Memory → Template → Skill Pattern

## Goal
Turn repeated work into reusable, privacy-safe automation building blocks.

## Stages

### Stage 1: Private capture (raw)
Capture real inputs/outputs privately (do not publish). Example sources:
- inbox messages
- meeting notes
- user requests

### Stage 2: Abstraction (template)
Extract reusable structure:
- what the input looks like (schema)
- what the output must contain (checklist)
- decision rules

### Stage 3: Packaging (skill)
Create a skill skeleton:
- Purpose
- Inputs
- Outputs
- Procedure
- Failure modes
- Examples (synthetic)

## Rule of thumb
Promote to a skill when:
- you repeated it ≥3 times, or
- the cost of a mistake is high, or
- the steps are long and easy to forget.
